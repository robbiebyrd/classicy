# ClassicyWindow `backgroundColor` Prop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an optional `backgroundColor` prop to `ClassicyWindow` that sets the content area's background for both standard and modal windows.

**Architecture:** The prop is forwarded to the window contents `<div>` as an inline CSS custom property (`--classicy-window-contents-bg`, following the existing `--classicy-*` inline-variable convention used by ClassicyIcon and ClassicyColorPicker). The SCSS `background-color` declarations resolve that variable with the current theme defaults as fallback, so the modal rule's `!important` still honors the prop and absent-prop behavior is byte-identical to today.

**Tech Stack:** React 18 + TypeScript, SCSS, Vitest + Testing Library.

			backgroundColor: "#00ff00",

			".classicyWindowContentsModal",
		) as HTMLElement;
		expect(modalContents).not.toBeNull();
		expect(
			modalContents.style.getPropertyValue("--classicy-window-contents-bg"),
		).toBe("#00ff00");
	});
});
